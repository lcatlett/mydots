# Professional Formatting and Language Standardization Prompt

## Mission
Transform any artifact, document, or script to use professional, executive-appropriate language and consistent formatting standards suitable for business stakeholders and technical documentation.

## Language Transformation Guidelines

### 1. Remove Emotional and Inflammatory Language
**Replace:**
- CATASTROPHIC, CRITICAL, DISASTER → Significant, High-priority, Issue
- FAILURE, BROKEN, WRONG → Suboptimal, Requires optimization, Alternative approach
- STUCK, DEAD, KILLED → Long-running, Inactive, Terminated
- WASTE, BLOAT → Inefficient allocation, Increased utilization

**Examples:**
- "CATASTROPHIC FAILURE" → "Significant performance issue"
- "BROKEN ARCHITECTURE" → "Architecture requiring optimization"
- "WASTED RESOURCES" → "Suboptimal resource allocation"

### 2. Convert Caps Lock to Professional Case
**Replace:**
- ALL CAPS HEADERS → Title Case Headers
- SHOUTING TEXT → Normal sentence case
- URGENT/CRITICAL → High-priority/Significant

**Examples:**
- "URGENT ACTION REQUIRED" → "High-priority action required"
- "SYSTEM FAILURE DETECTED" → "System performance issue detected"

### 3. Remove Emojis and Symbols
**Replace:**
- 🚨 ⚠️ ❌ → Remove entirely or replace with "Note:", "Alert:", "Issue:"
- ✅ ☑️ → "Completed:", "Verified:", or remove
- 🔍 📊 💰 → Remove entirely

**Examples:**
- "🚨 CRITICAL ISSUE" → "Critical issue identified"
- "✅ Task completed" → "Task completed"
- "📊 Data shows" → "Analysis indicates"

### 4. Convert Questions to Statements
**Replace interrogative with declarative:**
- "How does X affect Y?" → "Analysis of X impact on Y"
- "Why is this happening?" → "Root cause analysis indicates"
- "What should we do?" → "Recommended actions include"

### 5. Professional Terminology Substitutions
**Technical Terms:**
- "Stuck workers" → "Long-running processes" or "Extended processing workers"
- "Dead processes" → "Inactive processes" or "Terminated processes"
- "Killing processes" → "Terminating processes" or "Process cleanup"
- "Crashed" → "Stopped unexpectedly" or "Service interruption"

**Business Terms:**
- "Revenue at risk" → "Revenue exposure" or "Potential revenue impact"
- "Customer churn" → "Customer retention risk" or "Service continuity concern"
- "Bleeding money" → "Increased operational costs" or "Cost inefficiency"

### 6. Formatting Standardization

#### Headers and Sections
```
# Main Title (Title Case)
## Section Header (Title Case)
### Subsection Header (Title Case)
```

#### Lists and Bullets
- Use consistent bullet formatting
- Remove excessive punctuation
- Maintain parallel structure

#### Code and Commands
- Use proper code blocks with language specification
- Consistent indentation (2 or 4 spaces)
- Professional variable naming

#### Data Presentation
- Use tables for structured data
- Consistent number formatting (decimals, percentages)
- Professional units and measurements

## Example Transformations

### Before (Unprofessional)
```
🚨 CRITICAL SYSTEM FAILURE DETECTED! 🚨

The PHP workers are COMPLETELY STUCK serving massive ZIP files for WEEKS! This is a CATASTROPHIC ARCHITECTURAL DISASTER that's BLEEDING MONEY and KILLING PERFORMANCE!

❌ BROKEN: Static files through PHP (WRONG!)
✅ CORRECT: Direct file serving (RIGHT!)

WHY IS THIS HAPPENING?
- STUPID configuration mistakes
- NO timeout protection
- WASTED worker pools
```

### After (Professional)
```
System Performance Issue Analysis

PHP worker processes are experiencing extended processing times while serving large static files, with some requests running for multiple weeks. This represents a significant architectural optimization opportunity affecting resource efficiency and service performance.

Current implementation: Static file serving through PHP workers
Recommended approach: Direct file serving through web server

Root cause analysis:
- Configuration requires optimization for static file handling
- Timeout protection needs implementation
- Worker pool allocation requires efficiency improvements
```
